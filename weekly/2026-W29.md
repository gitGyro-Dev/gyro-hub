## Applied Model

```text
Boundary / Negative Definition
↓
Trajectory Detection
↓
Stability Response
```

1. **Boundary / Negative Definition** — preventive definition of prohibited inputs, transitions, and boundary crossings.
2. **Trajectory Detection** — evaluation of post-login event sequences as a Trajectory rather than isolated events.
3. **Stability Response** — challenge, restriction, isolation, recovery, and other damage-limiting responses.

Central proposition:

```text
valid events != stable trajectory
```

Individually valid and authorized events may still form an unstable or impermissible Trajectory.

## Positioning

The applied paper compares the proposed model with:

- Continuous Authentication
- Risk-Based Authentication
- Adaptive Authentication
- Behavioral Biometrics
- SIEM
- UEBA
- Zero Trust Architecture
- General Anomaly Detection

The proposal does not replace these fields. Its distinctive focus is the Stability and Deviation of the complete post-login operation sequence.

## Gyro Logic Mathematical Formalization Study

A new research artifact was added:

```text
gyrologic/docs/17_Mathematical_Object_Candidate_Table_20260713.md
```

The document creates **Mathematical Object Candidate Table v0** for:

- Structure
- Slice
- Stability
- Difference / Δ
- Trajectory

Current leading candidate families are recorded as:

```text
Structure   = relational constrained possibility-bearing object
Slice       = Orientation- and Context-parameterized partial process
Stability   = structured continuing-establishment condition
Difference  = Slice-relative directed discrepancy
Trajectory  = history-bearing indexed composition of Core realizations
```

These are research candidates, not final definitions. The document explicitly preserves:

```text
Structure → Slice → Stability
```

and records failure warnings against premature reductions such as `Structure = set`, `Slice = total function`, `Stability = scalar`, `Δ = metric`, or `Trajectory = Markov state sequence`.
