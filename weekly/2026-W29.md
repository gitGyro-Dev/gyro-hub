# Gyro Weekly 2026-W29

> GyroAuth Trajectory-Based Vulnerability Response publication milestone.

## Overview

The English and Japanese versions of the GyroAuth applied paper on post-login trajectory monitoring and vulnerability response were published on Jxiv.

This work extends GyroAuth from login-time authentication toward post-login monitoring by evaluating the Stability and Deviation of an operation Trajectory rather than treating individually valid events as sufficient evidence of safety.

## Main Model

```text
Boundary / Negative Definition
↓
Trajectory Detection
↓
Stability Response
```

1. **Boundary / Negative Definition** — preventive definition of prohibited inputs, transitions, and boundary crossings.
2. **Trajectory Detection** — evaluation of post-login event sequences as a Trajectory rather than isolated events.
3. **Stability Response** — challenge, restriction, isolation, recovery, and other damage-limiting responses.

Central proposition:

```text
valid events != stable trajectory
```

Individually valid and authorized events may still form an unstable or impermissible Trajectory.

## Positioning

The paper compares the proposed model with:

- Continuous Authentication
- Risk-Based Authentication
- Adaptive Authentication
- Behavioral Biometrics
- SIEM
- UEBA
- Zero Trust Architecture
- General Anomaly Detection

The proposal does not replace these fields. Its distinctive focus is the Stability and Deviation of the complete post-login operation sequence.

## PoC / Prototype

The minimal PoC evaluates three session types:

| Session | State | Response |
|---|---|---|
| normal | AUTH_STABLE | continue |
| suspicious | AUTH_CHALLENGE | challenge |
| attack-like | AUTH_ISOLATED | isolate |
