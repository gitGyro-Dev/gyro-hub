# Gyro Weekly

Gyro Weekly is a lightweight operating report for the Gyro ecosystem.

It is separate from the dashboard.

```text
Dashboard = current position
Weekly    = weekly flow
Roadmap   = future direction
```

## Reports

| Week | Report | Focus |
|---|---|---|
| 2026-W27 | [Gyro Weekly 2026-W27](weekly/2026-W27.md) | Gyro Hub, Project Cycle, data directory, first weekly log |
| 2026-W28 | [Gyro Weekly 2026-W28](weekly/2026-W28.md) | Gyro Logic v3.1, Core Definitions, Boundary integration, paper preparation |

## Purpose

Gyro Weekly records the movement of the project over time.

It should summarize what changed, what was published, what was prototyped, what was discussed, and what should happen next.

## Standard Sections

```text
# Gyro Weekly YYYY-Www

## Overview

## GitHub Updates

## Papers / Publications

## PoC / Prototype

## X / Communication

## Brainstorm / New Ideas

## Issues / Feedback

## Decisions

## Next Week

## Open Questions
```

## Future Automation

Gyro Weekly should eventually be generated by Gyro Commands on top of Gyro Developer Toolkit.

Possible command:

```text
python tools/gyro/weekly.py --week YYYY-Www
```

Expected outputs:

```text
weekly/YYYY-Www.md
data/weekly_index.json
```

## Principle

Weekly reports should not replace source documents. They should point back to GitHub, Google Drive, NotebookLM, X, Jxiv, Zenodo, and other source locations.
